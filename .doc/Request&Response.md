# 请求数据包和响应数据包解析

## 连接

由底层自动处理

## 断连

由底层自动处理

## 创建房间（CREATE）

- 请求数据包

  ```
  [标记位:1][房间名长度n:1][房间名:n]
  ```

  房间名应该是客户端的用户名

- 响应数据包

  ```
  [标记位:1][负载长度:2][ID内容:4]
  ```

## 查询房间（QUERY）

- 请求数据包

  ```
  [标记位:1]
  ```

- 响应数据包

  ```
  [标记位:1][负载长度:2] [[count:1] + count*([roomId:4][flags:1][nameLen:1][nameBytes])]
  ```

  如果长度超过short类型最大值应该抛错

## 加入房间（JOIN）

- 请求数据包

  ```
  [标记位:1][房间id:4]
  ```

- 响应数据包

  - 当房间不存在时

    ```
    [标记位:1][负载长度:2][成功标志位:1][房间id:4]
    ```

    加入失败时成功标志位和房间id均为0

  - 当房间已开始游戏时

    ```
    [标记位:1][负载长度:2][失败原因代码:1]
    ```

  - 当房间已满时

    ```
    [标记位:1][负载长度:2][失败原因代码:1]
    ```

  - 成功加入时

    - 对于房客

      ```
      [标记位:1][负载长度:2][成功标志位:1][房间id:4]
      ```

      成功标志位置为1

    - 对于房主（当存在时）

      ```
      [标记位:1][负载长度:2][房间id:4]
      ```

      相当于是重新推送了一遍确认信息

## 开始游戏（START）

- 请求数据包

  ```
  [标志位:1]
  ```

- 响应数据包

  - 非房主或当前房间不存在时

    ```
    [标记位:1][负载长度:2][NOT_HOST代码:1]
    ```

  - 当前房间未满时

    ```
    [标记位:1][负载长度:2][NOT_READY代码:1]
    ```

  - 成功开始游戏

    双方都推以下信息

    ```
    [标记位:1][负载长度:2]
    ```

    负载长度为0

## 房主退出房间（EXIT_ROOM）

- 请求数据包

  ```
  [标记位:1]
  ```

- 响应数据包

  - 错误处理：略

  - 退出房间成功

    - 对于房客（如果存在）

      ```
      [标志位:1][负载长度:2]
      ```

    - 对于房主：同房客

## 房客退出房间（LEAVE_ROOM）

- 请求数据包

  ```
  [标志位:1]
  ```

- 响应数据包

  - 错误处理：略

  - 离开房间成功

    - 对于房客：

      ```
      [标志位:1][负载长度:2]
      ```

    - 对于房主：

      ```
      [标志位:1][负载长度:2][房间id:4]
      ```

      给房主推送一遍他的房间的房客离开了

## 离开（LEAVE）

保留
